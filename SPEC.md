> 本ドキュメントは Claude Code がこのプロジェクトを実装する際の参照仕様書です。
> プロジェクトルートに `SPEC.md` として配置し、`CLAUDE.md` および各エージェント定義から参照します。

## 1. プロジェクト概要

従来のテトリスに「上下双方向からの落下」と「中央へ向かう重力」という新しい物理法則を組み込んだスタンドアロン・パズルゲーム。

特徴的なゲームメカニクス:

- 盤面の中央に基準層が存在し、ブロックは上端からも下端からも出現する
- ブロックは常に「中央方向」に向かって落下/上昇する
- 中央境界線が一気にライン消去されると、上下のブロックが**同時に中央へ崩落**して**大連鎖トリガー**となる

### 1.1 ターゲット

- 単一ユーザーのローカル実行
- スタンドアロンの JAR として配布可能
- マウス不要、キーボード操作のみ

### 1.2 完成の定義

- 3 つのゲームモード(ランダム/交互/ユーザー選択)がすべて動作する
- 中央境界線の同時消去ボーナスが発火する
- ハイスコアがローカルに保存される
- 1 回もクラッシュせずに最低 10 分間プレイできる

## 3. 盤面仕様

### 3.1 サイズと座標系

- **幅**: 10 セル (col 0 ~ 9)
- **高さ**: 21 セル (row 0 ~ 20)
- **中央行 (Center Line)**: row 10
- **上半分**: row 0 ~ 9
- **下半分**: row 11 ~ 20
- **座標原点**: 左上 (0, 0)
- **row 増加方向**: 下向き

```
row 0  ── 上端(上から出現するブロックのスポーン)
row 1
 ...
row 9  ── 上半分の最下行
row 10 ── 中央境界線
row 11 ── 下半分の最上行
 ...
row 19
row 20 ── 下端(下から出現するブロックのスポーン)
```

### 3.2 セルの状態

各セルは以下のいずれか:

- `EMPTY` : 空
- `FILLED(color)` : 確定したブロックで埋まっている(色情報を持つ)

落下中のテトリミノは盤面とは別管理(オーバーレイ)。

## 4. テトリミノ仕様

### 4.1 種類

標準 7 種類: I, O, T, S, Z, J, L

各ミノは色と回転状態(0°, 90°, 180°, 270°)を持つ。

### 4.2 出現(スポーン)

| 出現方向 | 初期位置 | 初期回転 |
|---------|---------|---------|
| 上から落下 (DOWN) | row 0~1, col 3~6 を中心 | 0°(標準) |
| 下から上昇 (UP) | row 19~20, col 3~6 を中心 | 180°(上下反転) |

下から出現するミノを 180° 回転させるのは、視覚的にミノが「正しい向きで現れて、その向きのまま中央へ進む」ように見せるため。

### 4.3 回転システム

- **SRS (Super Rotation System)** を採用
- 壁蹴り (Wall Kick) のオフセットテーブルは公式 SRS 準拠
- 上向き重力のミノには、SRS のオフセットを上下反転して適用

### 4.4 落下/上昇方向

ミノは出現方向によって以下のように移動する。

- **DOWN ミノ**: 中央(row 10)に向かって row が増加する方向に移動
- **UP ミノ**: 中央(row 10)に向かって row が減少する方向に移動

## 5. ゲームメカニクス

### 5.1 ブロックの固定 (Lock)

ミノが次の移動先に進めない場合(他のブロックに衝突 or 中央境界線を越えようとした場合)、現在位置でロック(固定)される。

- DOWN ミノは row 10 を越えて row 11 以下には侵入できない
- UP ミノは row 10 を越えて row 9 以上には侵入できない
- 中央境界線(row 10)はミノが配置可能だが、UP と DOWN で同じセルを取り合う場合は先着順

### 5.2 ライン消去 (Line Clear)

ブロックがロックされた直後に判定する。

#### 5.2.1 判定範囲

- **上半分の消去**: row 0~9 のうち、横一列が `FILLED` になっている行
- **下半分の消去**: row 11~20 のうち、横一列が `FILLED` になっている行
- **中央境界線の消去**: row 10 が横一列 `FILLED` の場合(特殊扱い)

#### 5.2.2 崩落 (Cascade) の挙動

消去された行を削除した後、残ったブロックは「中央方向」に向かって 1 行ずつ詰める。

- 上半分の消去後: 消えた行より上(row 番号が小さい側)のブロック群を中央へ向けて 1 行ずつ下にシフト
- 下半分の消去後: 消えた行より下(row 番号が大きい側)のブロック群を中央へ向けて 1 行ずつ上にシフト
- 中央境界線の消去後:
  - 上半分のブロック群がすべて 1 行下にシフト
  - **同時に**下半分のブロック群がすべて 1 行上にシフト
  - 両者が中央で噛み合う形になる(大連鎖の起点)

#### 5.2.3 連鎖判定

シフト完了後、再度ライン消去判定を行う。ラインが揃えば再度消去 → シフト → 判定…を繰り返す。

連鎖回数(REN)をカウントし、スコアボーナスに反映する。

### 5.3 ゲームオーバー

**主条件**: 「次のミノを出現させる際、スポーン位置に既にブロックがあって配置できない」状態でゲームオーバー。

具体的には:

- row 0~1 のスポーン領域がブロックで埋まっていて DOWN ミノを配置できない
- row 19~20 のスポーン領域がブロックで埋まっていて UP ミノを配置できない

どちらか一方でも該当した時点でゲームオーバー。

## 6. ゲームモード

### 6.1 ランダムモード

次に出現するミノの方向(UP / DOWN)が完全ランダム。バッグ生成時に方向もランダムに割り当てる。

### 6.2 交互モード

DOWN → UP → DOWN → UP… と完全に交互に出現。初手は DOWN 固定。

### 6.3 ユーザー選択モード(メイン)

プレイヤーが Q キー(上から落下)または E キー(下から上昇)を押して、次のミノの方向を選択する。

- 選択タイミング: 前のミノがロックされた瞬間から、次のミノが盤面に現れて 0.5 秒以内(猶予時間)
- 猶予時間内に選択しなかった場合: 直前と同じ方向で自動出現
- 落下中の方向反転は不可

## 7. Hold 機能と NEXT キュー

### 7.1 NEXT キュー

- 上下で共通の 1 系列のキュー
- 標準 7-bag アルゴリズム(7 種ランダムシャッフルの繰り返し)で生成
- 表示数: 次の 3 個を表示

### 7.2 Hold

- Hold 枠は 1 つのみ(上下で共有)
- Hold 操作キー(C または Shift)で現在のミノを Hold へ送る
- Hold 時、ミノの方向属性(UP/DOWN)は破棄され、種類(I, O, T, ...)のみが保存される
- Hold から取り出す際:
  - ランダムモード: 取り出した時点でランダムに方向決定
  - 交互モード: その時点の交互順序に従う
  - ユーザー選択モード: プレイヤーが Q/E で方向を選択
- 1 ミノにつき Hold は 1 回まで(連続 Hold 禁止、Tetris 標準ルール)

## 8. 操作仕様

### 8.1 デフォルトキーバインド

| 操作 | キー |
|------|------|
| 左移動 | ← |
| 右移動 | → |
| ソフトドロップ(中央方向に加速) | ↓ |
| ハードドロップ(即座にロック) | Space |
| 時計回り回転 | ↑ または X |
| 反時計回り回転 | Z |
| Hold | C または Shift |
| 方向選択: 上から | W |
| 方向選択: 下から | S |
| ポーズ | Esc または P |

方向選択キーは標準テトリスのキーバインドにない双方向テトリス独自の操作。X(時計回り回転)から同じ縦列を上に滑らせるだけで届く位置(W=2 行上、S=1 行上)を採用し、WASD の上下メタファー(W=上、S=下)で意味を覚えられるようにしている。設計判断の経緯は `docs/decisions/0001-direction-selection-keys.md` を参照。

### 8.2 入力受付

- Lock Delay: 0.5 秒(中央方向への移動が止まってから固定されるまでの猶予)
- DAS (Delayed Auto Shift): 167ms
- ARR (Auto Repeat Rate): 33ms

## 9. スコアリング

### 9.1 基本得点(レベル 1 換算)

| 消去ライン数 | 得点 |
|------------|------|
| 1 ライン (Single) | 100 |
| 2 ライン (Double) | 300 |
| 3 ライン (Triple) | 500 |
| 4 ライン (Tetris) | 800 |

### 9.2 特殊ボーナス

| ボーナス | 倍率 |
|--------|------|
| 中央境界線(row 10)消去 | **基本得点 × 2** |
| 上下同時消去(中央境界線消去で上下が同時に揃った) | **基本得点 × 5** |
| 連鎖 (REN) | 連鎖数 × 50 を加算 |
| Back-to-Back Tetris | Tetris 得点 × 1.5 |
| ハードドロップ | 落下距離 × 2 加算 |
| ソフトドロップ | 落下距離 × 1 加算 |

### 9.3 レベルと速度

- レベル 1 開始
- 10 ライン消去ごとにレベルアップ
- レベルアップで落下速度が増加(レベル 1 で約 1.0 秒/セル、レベル 20 で約 0.016 秒/セル)
- レベル別の基本得点倍率: `基本得点 × レベル`


### 12.2 レイヤ依存

- `domain` は他のどのパッケージにも依存しない(純粋な値オブジェクトとロジック)
- `game` は `domain` のみに依存
- `ui` は `game` と `domain` に依存
- `input` は `game` に依存
- `persistence` は `domain` のみに依存

UI 層を入れ替えやすくする(将来 Swing や他の描画系に移行できる)。

### 12.3 ゲームループ

JavaFX の `AnimationTimer` を使用:

```
[60 FPS で]
  1. 入力を消化(KeyboardController から GameEngine へコマンド送信)
  2. GameEngine.tick() を呼び出し、内部状態を更新
     - 落下/上昇タイマー処理
     - Lock Delay 処理
     - ライン消去判定
     - スコア更新
  3. GameCanvas.render(gameState) を呼び出し、画面を再描画
```

## 14. コーディング規約

### 14.1 命名

- パッケージ: 小文字単数形
- クラス: PascalCase
- メソッド: camelCase
- 定数: SCREAMING_SNAKE_CASE
- 列挙型: PascalCase

### 14.2 設計指針

- **不変オブジェクト優先**: Position, Tetromino, Cell など値オブジェクトは `final` で immutable
- **副作用の局所化**: GameEngine 内で状態変更、それ以外は純粋関数的に
- **UI とロジックの分離**: domain と game は JavaFX に依存させない
- **テスト容易性**: ロジック層はすべて単体テスト可能な形に
- **マジックナンバー禁止**: 定数として名前を付ける(`BOARD_WIDTH = 10` 等)

### 14.3 テスト

- `domain/`, `game/` の各クラスに単体テスト必須
- 特に LineClearService、CollisionDetector、RotationSystem は網羅的なテスト
- UI 層のテストは目視確認で OK(自動化しない)

### 14.4 例外処理

- ゲームロジック内で起きうる例外は明示的に処理
- UI 層では try-catch で囲み、ユーザーにエラーダイアログを表示
- ログには SLF4J を使用

## 16. やってはいけないこと

- マジックナンバーをコードに直接書く(必ず定数化)
- UI 層からドメインロジックを呼ばずに直接盤面を変更する(GameEngine 経由)
