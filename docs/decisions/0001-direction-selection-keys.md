# ADR 0001: 方向選択キーを Q/E から W/S に変更

- **状態**: 採用
- **日付**: 2026-05-14
- **適用範囲**: Phase 4 で実装される SPEC §6.3 ユーザー選択モード

## 背景

SPEC §8.1 の初版は「上から落下」(Q) と「下から上昇」(E) を方向選択キーに割り当てていた。これは双方向テトリス独自の操作であり、Tetris Guideline には対応する標準キーが存在しない。

プレイヤーから「従来のテトリスに似たキーバインドを維持しつつ、追加の操作は手の構造的に押しやすいキーにしてほしい」という要望を受けて再検討した。

## 前提条件: 標準テトリスのプレイ姿勢

- 右手: ← / → / ↓ / ↑ (移動 + 回転 + ソフトドロップ)
- 左手: Z (反時計回り) / X (時計回り) / C (Hold) / Space (ハードドロップ、親指)

左手は X 列(中指)と C 列(薬指/中指)を中心としたホームポジションを取る。

## 候補

| 候補 | 利点 | 欠点 |
|------|------|------|
| Q / E (SPEC 初版) | 上下対称、左右に並ぶ | 左手中指 (Q) と人差し指 (E) で 2 行上の段にホームポジションを大きく離れる必要がある |
| A / S | A・S とも上段 1 行差、薬指で連続 | A は小指を Z から離して左に動かす必要、ホームポジションを崩す |
| W / S | WASD の上下メタファー、X からの縦移動だけで届く | W は X から 2 行上で若干遠い |
| D / F | 両方ホーム段、人差し指+中指で押下しやすい | 「上下」のメタファーがなく覚えにくい |
| S / D | 両方ホーム段、最も押しやすい | 上下のマッピングが任意で覚えにくい |

## 採用

**W = 上から落下 (DOWN 方向のミノ)、S = 下から上昇 (UP 方向のミノ)** を採用する。

理由:

1. **動的意味の覚えやすさ**: WASD の慣習で W=上、S=下 が広く共有されている。「上から」「下から」のスポーン位置の上下と一致する
2. **指の動線**: 時計回り回転キー X から同じ縦列を上に滑らせるだけ。指の入れ替えが不要
3. **干渉が少ない**: 標準キーバインド(矢印・Z/X/C/Shift/Space/Esc/P)と物理的に分離。誤操作のリスクが低い
4. **W が遠い件**: SPEC §6.3 の方向選択猶予は 500ms あり、瞬時の連打ではないので 2 行ぶんの移動はプレイ感に悪影響を与えにくい

## 影響

- SPEC §8.1 の表を更新(Q/E → W/S)
- Phase 4 実装時に `KeyBindings.DIRECTION_DOWN_SPAWN = {KeyCode.W}` と `DIRECTION_UP_SPAWN = {KeyCode.S}` を追加
- Phase 1〜3 のキーバインド(矢印・Z/X/Space/C/Shift/Esc/P/R)は無変更
