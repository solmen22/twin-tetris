# 双方向重力テトリス Web サービス化 ロードマップ

> 本ドキュメントは、ゲームプレイ本体(Phase 1〜5 完了済み)に続くメタ機能群の実装順序を定めるもの。
> 詳細な機能カタログは `docs/research/web-tetris-feature-survey.md` を参照。

---

## 背景

- Phase 5 までで双方向重力テトリスのコアゲームプレイは完成
- TeaVM により Web 版がビルドできる状態(`./gradlew webDist`)
- JavaFX デスクトップ版は維持(同一ソースツリーに並存)
- 今後は「個人が運営する Web サービス」として継続運用していく方針
- バックエンドは将来的に無料 BaaS(Supabase / Firebase 等)を採用予定

---

## 採用方針

### 全体原則

1. **クライアントだけで完結する機能を優先実装**(サーバ依存を後ろ倒し)
2. **ゲストプレイは常に可能**(アカウント無しでも一通り遊べる)
3. **不正対策はサーバ導入と同時に最低限を実装**(HMAC 署名 + サーバ側上限チェック)
4. **設定値は LocalStorage → 将来サーバ同期できる構造にしておく**
5. **JavaFX 版とは Web 固有機能(PWA, タッチ, ランキング等)で分岐**。共通ロジックは `domain/` `game/` を維持

### 採用 / 不採用の選択

| 領域 | 採用 | 不採用 / 後回し | 理由 |
|-----|-----|--------------|------|
| 認証方式 | ユーザー名+パスワード + 匿名先行(Supabase Auth) | OAuth, メール検証(Phase B+) | 初期コスト最小、必要になったら追加 |
| ランキング軸 | Marathon スコア / 中央崩落回数 の 2 軸 | モード別個別ランキング、期間別 | 分散させない |
| キーバインド | `KeyboardEvent.code` ベース | `key` ベース | レイアウト非依存にするため |
| チュートリアル | 静的ページ(Phase A) → インタラクティブ(Phase C) | 動画埋め込み | 段階的に強化 |
| 対戦モード | **非採用**(SPEC 非ゴール) | - | SPEC.md の方針に従う |
| 課金 | **非採用** | - | 個人運営、無料サービス前提 |
| 多言語化 | 日本語のみ(Phase A) → 英語追加(Phase C) | 中韓その他 | 段階的 |

## Phase A: 足回り(サーバ不要)

**目的**: 「ちゃんとした Web ゲーム」感を一気に出す。サーバ導入前に整えるべき土台。

| # | 機能 | 概要 | 影響範囲 |
|---|------|------|---------|
| A1 | **設定画面 + キーバインド** | 全アクションのキー再割り当て、DAS/ARR/SDF、音量、テーマ。LocalStorage 保存 | `web/`, `src/web/`, 新規 `SettingsStore` |
| A2 | **チュートリアル(静的)** | 「遊び方」ページ。GIF/アニメ + テキスト。中央境界線・方向選択を強調 | `src/web/guide.html` |
| A3 | **PWA 化** | `manifest.json` + Service Worker。ホーム画面追加・オフライン起動 | `src/web/`, ビルド出力に SW を含める |
| A4 | **モバイル/タッチ対応** | レスポンシブ + 仮想ゲームパッド(画面端の半透明ボタン) | `src/web/style.css`, タッチイベントハンドラ |
| A5 | **利用規約・プライバシーポリシー** | フッターに常時リンク。サーバ導入前に整備 | `src/web/terms.html`, `src/web/privacy.html` |
| A6 | **結果画面の詳細統計** | スコア・ライン・レベル・最高連鎖・中央崩落回数・プレイ時間 | `WebMain`, `HudView` |
| A7 | **ローカルベスト記録** | 既存の `bestScore` を「モード別 + 軸別」に拡張、LocalStorage 永続化 | `SettingsStore` に統合 |

着手順: A1 → A6 → A7 → A2 → A4 → A3 → A5

## Phase B: アカウント + ランキング(サーバ導入)

**目的**: 競技性とコミュニティの最小核を成立させる。

### バックエンド選定: **Supabase** を第一候補

理由:
- Postgres ベース(後で別 DB に移行しやすい)
- 匿名認証 → 正規アカウントへの昇格を標準サポート
- REST API + Row Level Security で TeaVM(Java)からも `fetch` で叩ける
- 無料枠が比較的緩い(500MB DB, 月 5GB bandwidth, 50k MAU)
- メール送信も標準サポート

代替候補: Firebase(JS SDK 前提のため TeaVM だと相性微妙)、Cloudflare D1 + Workers(自前認証必要)

> ⚠️ Supabase 採用は要ユーザー確認(外部依存追加のため)

| # | 機能 | 概要 |
|---|------|------|
| B1 | **匿名アカウント先行発行** | 初回アクセスで匿名 user_id を発行、LocalStorage に保存 |
| B2 | **正規アカウント登録 / ログイン** | ユーザー名 + パスワード(メールは任意) |
| B3 | **ハイスコア提出 API** | HMAC 署名 + サーバ側上限チェック(例: 1 秒 100 ラインは弾く) |
| B4 | **グローバルランキング表示** | Marathon / 中央崩落回数の 2 軸。自分の周辺順位も表示 |
| B5 | **プロフィールページ** | `/u/<username>`。個人ベスト、プレイ回数 |
| B6 | **設定のサーバ同期** | Phase A の SettingsStore を server-side mirror |

## Phase C: 体験を厚くする

| # | 機能 | 概要 |
|---|------|------|
| C1 | **リプレイ機能** | 入力イベント列 + 初期シードをサーバ保存、決定論的再生 |
| C2 | **リプレイ共有 URL** | `/replay/<id>` で他人も再生可能 |
| C3 | **統計ダッシュボード** | 上達曲線、モード別統計、中央崩落発火率 |
| C4 | **実績 / XP / レベル** | ゲーミフィケーション |
| C5 | **英語対応** | i18n フレームワーク導入、JSON 翻訳ファイル |
| C6 | **インタラクティブチュートリアル** | ステップ式 Lesson モード(PPT2 方式) |
| C7 | **アクセシビリティ強化** | 色覚モード、ハイコントラスト、キーボードナビゲーション、ARIA |
